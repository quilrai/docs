# CLAUDE.md

Read this before touching anything. This is the QuilrAI documentation site. It is a **Docusaurus 3.9.2** static site that builds to `build/` and auto-deploys to **GitHub Pages at https://docs.quilrai.dev** on every push to `main` (`.github/workflows/deploy.yml`). There is no app, no database, no backend here. It is docs. Treat content as production.

`AGENTS.md` is a symlink to this file. Same rules for every agent.

---

## Hard rules — break these and you break the build or the commit

1. **Every doc page and every category needs an icon, from the allowlist.** No exceptions.
   - `.md`/`.mdx` pages: frontmatter `sidebar_custom_props.icon: <Name>`.
   - `_category_.json`: `customProps.icon: <Name>`.
   - Valid names live in **`src/utils/sidebarIcons.js`** (the `iconMap`). That file is the single source of truth — not the list in `.cursor/rules/docs-icons.mdc`, which drifts.
   - If no existing icon fits: import a new one from `lucide-react` into `src/utils/sidebarIcons.js`, add it to `iconMap`, then use the exact export name. **`lucide-react` is v1.7.0 — brand icons were removed. `Github` and `Slack` DO NOT EXIST.** Verify an icon exists before importing: `node -e "console.log(!!require('lucide-react').Foo)"`.
   - Enforced by `.husky/pre-commit` → `node scripts/check-sidebar-icons.js --staged`. A missing/invalid icon **blocks the commit**. Run `npm run check-icons` to check everything.

2. **Broken internal links fail the build.** `onBrokenLinks: 'throw'` in `docusaurus.config.js`. Always run `npm run build` before you claim done. The build is the link checker.

3. **No em dashes in `docs/**/*.md`.** Use regular `-`. There's a cleanup script: `./replace-em-dashes.sh` (rewrites U+2014 → `-` across all docs). Don't introduce them in the first place.

4. **The sidebar is auto-generated from the filesystem.** `sidebars.js` is just `{type: 'autogenerated', dirName: '.'}`. You do NOT edit it to add pages. Ordering is controlled by `sidebar_position` (page frontmatter) and `position` (`_category_.json`).

5. **Docs are served at the root, not `/docs`.** `routeBasePath: '/'`. `docs/intro.md` has `slug: /` and is the homepage. Legacy `/docs/*` URLs are redirected (see `plugin-client-redirects` config). Internal links between docs are relative (e.g. `./mcp-provider-setup/overview`, `../onemcp`).

6. **`static/llms.txt` is hand-maintained and must mirror the docs.** When you add, remove, rename, or move a page, update its entry there (URLs use the `.md` suffix — those are served by the markdown plugin, see below). Stale entries = 404s for LLM consumers.

## Where everything is

### `docs/` — all content
Each top-level folder is a product area = one sidebar category, and needs a `_category_.json` (label, position, `generated-index` link, icon). The sections:

| Path | What it documents |
|------|-------------------|
| `docs/intro.md` | Homepage (`slug: /`). Has `<AskAiBanner>`. |
| `docs/llm-gateway/` | LLM Gateway: unified API for OpenAI/Anthropic/Azure/Bedrock/Vertex/Copilot. Quick start, architecture, integration, provider support, unified completions, Bedrock setup, HA/SLA, log export. `features/` = per-feature pages (routing, guardrails, intents, rate limits, prompt store, SDK mode, etc.). |
| `docs/mcp-gateway/` | MCP Gateway: manage/secure MCP servers for agents. Quick start, architecture, integration, OneMCP, log export. `features/` = per-feature pages. **`mcp-provider-setup/`** = per-provider connection setup (overview + slack/github/zoho); see below. |
| `docs/compliance-api-integration/claude/` | Anthropic Compliance API integration — monitor Claude.ai usage, DLP, key management. |
| `docs/endpoint-agent/` | Endpoint agent: `network-monitoring/` (TLS MITM, DLP) and `process-mapping/` (app discovery, enforcement). Each has quick-start + architecture. |
| `docs/roles-and-permissions/` | Labeled "QuilrAI Platform": `rbac/` (roles, screen permissions) and `integrations/data-forwarding/` (syslog, webhook). |
| `docs/playground/` | Interactive playground docs (uses `<SdkApiKeyTester>` / `<LogExportPlayground>`). |

`docs/mcp-gateway/mcp-provider-setup/` is the pattern for "add a provider": `overview.md` explains when you need an OAuth app vs. just a URL, plus shared prereqs/secret handling; `slack.md` and `github.md` are OAuth-app providers (create app → copy Client ID + Secret); `zoho.md` is a no-OAuth-app provider (generate a per-org MCP URL in the Zoho console, add it manually). To add another provider: drop in a new `.md` (with an icon), bump `sidebar_position`, add it to `overview.md`'s provider list and to `static/llms.txt`.

### `src/` — Docusaurus customization (this is a heavily themed site; don't casually rewrite theme files)
- **`src/theme/MDXComponents.js`** — registers components available in **every** `.md`/`.mdx` with **no import needed**:
  - `<StepFlow>` — horizontal step/flow diagram (`steps={[{label, items:[]}]}`; `✓`/`✗` in items get colored).
  - `<ArchitectureDiagram>` — pipeline/box diagram.
  - `<ExpandableTable>` — **overrides the default `table` element**, so every Markdown table is automatically expandable/fullscreen. You usually just write a normal Markdown table.
  - `<SdkApiKeyTester>`, `<LogExportPlayground>` — interactive playgrounds that call **live `*.quilr.ai` endpoints**. Used in `docs/playground/`.
- `src/theme/` (rest) — swizzled overrides: sidebar items (`DocSidebarItem/*` render the icons), code block copy buttons, doc cards, generated-index page, navbar + mobile sidebar, admonitions, TOC, search bar, layout. Changing these affects every page — be careful.
- `src/components/` — the components above plus `AskAiBanner`, `DocPageCopyDropdown` (the "copy page as markdown / open in ChatGPT/Claude" menu), `HomepageFeatures`, `SidebarThemeToggle`.
- **`src/utils/sidebarIcons.js`** — the icon allowlist (see rule 1).
- `src/data/aiProviders.js` — provider list + prompt builders for the "Ask AI" features.
- `src/themes/prismLight.js`, `prismDark.js` — code-block syntax themes.
- `src/css/custom.css` — global CSS (Tailwind v4 via `@tailwindcss/postcss`, + typography plugin).

### Everything else
- **`plugins/doc-page-markdown.js`** — serves every doc as a raw `.md` file (dev: middleware; prod: written to `build/`). Powers "copy/view as markdown" and the `.md` URLs in `llms.txt`. If a page exists, `<page>.md` exists.
- **`scripts/check-sidebar-icons.js`** — the icon enforcer (rule 1). Runs on pre-commit (`--staged`) and via `npm run check-icons` (all).
- **`static/`** — copied to site root verbatim. `img/` (referenced in docs as `/img/...`), `llms.txt` (rule 6), `CNAME` (custom domain), `robots.txt`.
- `.cursor/rules/docs-icons.mdc` — Cursor rule restating the icon requirement.
- `.github/workflows/` — `deploy.yml` (build + deploy to GH Pages on push to `main`), `test-deploy.yml` (PR build check).
- `sidebars.js` (autogenerated — don't edit for content), `docusaurus.config.js`, `package.json`.
- `build/`, `.docusaurus/`, `node_modules/` — generated. Never edit, never commit by hand.

## Commands

```bash
npm start            # local dev server with hot reload
npm run build        # production build — ALSO your link + config validator. Run before done.
npm run serve        # serve the built site locally
npm run check-icons  # validate all sidebar icons (subset of pre-commit)
./replace-em-dashes.sh   # strip em dashes from docs/**/*.md
```

Node >= 20.

## Writing a page — the checklist
1. Create `docs/<section>/<page>.md`. New top-level section also needs `_category_.json` (label, position, icon, usually a `generated-index` link).
2. Frontmatter, every page:
   ```yaml
   ---
   sidebar_position: <n>
   sidebar_custom_props:
     icon: <NameFromSidebarIcons>
   ---
   ```
3. Write content. Tables are auto-expandable. Use `<StepFlow>` / `<ArchitectureDiagram>` for flows (no import). Admonitions (`:::tip`, `:::note`, `:::warning`) are themed. Mermaid is enabled. Images go in `static/img/`, referenced as `/img/<file>`.
4. Internal links are relative and have no `.md` extension in the rendered link (e.g. `[OneMCP](../onemcp)`).
5. No em dashes.
6. If this page should be discoverable by LLMs, add it to `static/llms.txt`.
7. `npm run build` — confirm no broken-link or icon errors.

When in doubt, copy the structure of an existing neighboring page. Match the surrounding tone and frontmatter exactly.
